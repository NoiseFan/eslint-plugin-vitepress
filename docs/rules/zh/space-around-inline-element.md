# 行内元素周围空格

保持 Markdown 行内元素周围的空格风格一致。

这条规则主要用于处理正文中的 `link`、`image`、`inlineCode`、`emphasis` 和 `strong` 节点，让它们与前后文本、标点之间的间距更稳定，避免出现中英文混排时忽有空格、忽无空格的情况。

## 规则详情

默认情况下，这条规则要求被检查的行内元素与前后的普通文本之间使用且只使用一个空格。

该规则只检查以下节点：`link`、`image`、`inlineCode`、`emphasis`、`strong`

以下内容不在检查范围内：引用式链接和引用式图片、HTML 行内节点、普通文本节点、硬换行节点。

除了普通文本场景外，这条规则还会根据相邻字符是否为标点来调整空格要求：

- 行内元素前面是全角标点、左括号一类的成对起始标点，或 `/` 时，前面不应有空格
- 行内元素前面是半角标点时，前面必须保留一个空格
- 行内元素后面是全角标点、英文逗号、句号、右括号一类的结束标点时，后面不应有空格
- 行内元素后面是英文破折号或半角左括号时，后面必须保留一个空格

:::tip
1. 相邻的两个受检查行内元素会按“中间只保留一个空格”的方式处理，例如：``**命令行参数:** `--browser.ui```
2. 表格单元格中的首尾空白不会被这条规则处理，但单元格内部相邻的多个行内元素之间仍然会检查空格。
3. 末尾标题锚点和 VitePress 自定义容器标记等特殊场景会跳过，避免把内容错误地并到下一行或锚点上。
:::

该规则支持自动修复。

## 不适用场景

如果你的项目不希望统一 Markdown 行内元素与前后文本的空格风格，或者你希望把这类间距问题完全交给其他格式化工具处理，可以关闭这条规则。
